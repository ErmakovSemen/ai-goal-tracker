1) JSON тренера (Trainer JSON)
Назначение
Описывает характер, правила речи, запреты, обязательные слова и готовые реплики для конкретного тренера.
Основные поля
id — идентификатор тренера (strict / normal / gentle)
behavior — стиль общения
tone — тип стиля
style_rules — правила речи
stop_words — запрещённые слова
must_use — желательные слова
emoji_level — уровень эмодзи
prompt_rules — правила для промпта
language — язык
response_format — формат ответа
must_include_actions — обязательные действия
forbidden_actions — запрещённые действия
phrases — готовые реплики для ключевых событий
success_on_time — выполнено в срок
success_ahead — выполнено заранее
missed_deadline — дедлайн сорван
Зачем нужен
Позволяет тренеру быть отдельной “персоной”, а не просто текстом.
Один и тот же результат звучит по‑разному у разных тренеров.

2) JSON пола (Gender JSON)
Назначение
Хранит грамматические формы речи и местоимения для корректного текста.
Основные поля
gender — male или female
pronouns — местоимения по ролям
speech.forms — словарь форм слов (готов/готова, сказал/сказала и т.д.)
prompt_hint — подсказка для промпта “говори от женского/мужского лица”
Зачем нужен
Позволяет переключать пол тренера, не переписывая фразы вручную.

Что НЕ входит в Trainer JSON
(по нашему решению)
Ассеты (картинки/rig) — отдельная структура
Варианты пола — отдельный JSON пола
Интервалы уведомлений — хранятся отдельно (логика/БД), в JSON тренера остаются только фразы


Глоссарий (кратко)
Trainer JSON — профиль тренера: тон, правила речи, словари, реплики.
Gender JSON — набор форм речи для пола (готов/готова и т.д.).
Behavior — блок, который задаёт стиль и правила общения тренера.
Prompt rules — технические правила для ответа модели.
Phrases — готовые реплики под основные события (успех/провал).
Stop words — слова, которые ломают стиль (запрещены).
Must use — слова, которые усиливают стиль (желательны).
Assets — визуальные материалы (PNG/rig), не входят в Trainer JSON.
Proactive — логика уведомлений (интервалы хранятся отдельно).
