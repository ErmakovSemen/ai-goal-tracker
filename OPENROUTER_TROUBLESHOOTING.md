# Решение проблем с OpenRouter

## Текущая проблема: "No cookie auth credentials found"

Это означает, что OpenRouter не может аутентифицировать запрос.

## Возможные решения:

### 1. Проверьте формат ключа

Убедитесь, что ключ скопирован полностью и начинается с `v1-` или `sk-or-v1-`.

### 2. Проверьте ключ на сайте OpenRouter

1. Перейдите на https://openrouter.ai/keys
2. Убедитесь, что ключ активен
3. Скопируйте ключ заново (возможно, была ошибка при копировании)

### 3. Попробуйте создать новый ключ

1. На https://openrouter.ai/keys
2. Удалите старый ключ
3. Создайте новый
4. Скопируйте его в `.env`

### 4. Проверьте, что ключ правильный

Попробуйте протестировать ключ напрямую:

```bash
curl -X POST "https://openrouter.ai/api/v1/chat/completions" \
  -H "Authorization: Bearer YOUR_KEY_HERE" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "meta-llama/llama-3.1-8b-instruct",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

Если это не работает, ключ неверный или неактивен.

### 5. Альтернатива: Используйте бесплатные модели через другой провайдер

Если OpenRouter не работает, можно попробовать:
- Groq (если получится зарегистрироваться)
- Hugging Face (с исправленным API)
- Локальные модели через Ollama

## Текущий статус

Код для OpenRouter настроен правильно. Проблема в аутентификации - нужно проверить ключ.
